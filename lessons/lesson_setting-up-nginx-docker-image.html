Setting Up Docker NGINX

So first off, this is really simple...now... And that is the beauty of using docker images, they can speed up just about anything! Docker has some really great walk-throughs and tutorials and if you are looking for resources to learn more about Docker here are some great ones.

Getting started, I'm going to assume you have Docker installed, if not check out the instructions here: Getting Started with Docker Once Docker is installed just follow the steps below.

Step 1

Getting the NGINX Docker Image

You need a copy of the NGINX Docker image on your local environment. You can achieve this by inputting the following command. This will pull the latest/newest NGINX image in the Docker Repo.

docker pull nginx

Step 2

Run the Docker Image

Now that you have image you can simply run the image, passing in the working directory where your website is located to the NGINX website directory. This allows you to simply navigate to http://localhost and boom see changes live as you make them. Navigate to the directory in your terminal where you website id located, and run this command.

MacOS and Probably Linux

docker run --rm -v $(pwd):/usr/share/nginx/html -p 80:80 nginx

Windows 10

docker run --rm -v %cd%:/usr/share/nginx/html -p 80:80 nginx

What is this command doing?

docker run - tells docker to run an image
--rm
-v is doing the magic of mapping the current directory you are in (your website directory) to the websites directory of the docker image
-p is mapping port 80 on the NGINX container to your host, allowing you to browse to http://localhost:80 to see your site.
nginx is the image you want to run (this is the image you pulled down in step 1)
